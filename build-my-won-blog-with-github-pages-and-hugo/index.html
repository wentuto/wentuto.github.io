用 GITHUB PAGES + HUGO 打造我自己的BLOG

December 6, 2023 in BLOG

前言（為何使用 GITHUB PAGES)

之前已經使用過 GHOST 寫自己的BLOG，它是一套非常好用的 open source blog，功能相當強大，有很好用的編輯介面與擴充功能，但也因功能強大，管理邏輯上較複雜，遇到問題要克服也較麻煩，常擔心升級失敗後，之前的心血就白費了。我自己則是在更新 HTTPS 的 SSL (Let’s Encrypt)遇到問題後，就沒有再繼續更新自己舊的 GHOST BLOG。另一個原因是使用 GHOST 會有一筆 hosting 的費用（自己 host 網站或者是購買 GHOST 的 services），當知道可以用 GITHUB PAGES 來 publish 自己的靜態網頁，就想要試試看這個免費的方式。

評估（決定使用 HUGO 當靜態網頁生成器）

若要使用 GITHUB PAGES publish 網頁，需要先決定靜態網頁生成器的方案，它的作用是將你寫的 markdown 文章轉成網頁，你就可以將轉好的文章網頁上傳到 GITHUB PAGES 上去發表。靜態網頁生成器的方案有像 jekyll, hexo, and hugo。 比較後決定使用 HUGO，主要是『快』：因為 HUGO 是目前調查的靜態網頁系統中生成速度最快的，可以節省不少生成的時間。而 HUGO + GITHUB PAGES 這樣的方式比起之前 GHOST 的方案來說相依的模組較少，執行起來相對『簡單』很多，想說等之後真的越寫越多，再搬到 GHOST 也不是不可以 XD。

安裝

首先是在電腦上安裝 HUGO 這個用 Go 編寫的靜態網站生成器，由於我用的電腦是 MACBOOK，所以使用下面 command

brew install hugo

這樣就可以安裝完成了。

使用 HUGO 建立新 BLOG

至於怎麼使用 HUGO，我是參考這個 Hugo 貼身打造個人部落格 來入門，以及參考網路上的其他文章，一步步地寫 MARKDOWN，並找到方法加入 theme、圖片、程式碼、上傳 GITHUB…，這裡只列出如何用 HUGO 寫文章的基本方式，其他的有機會在慢慢來寫 XD！

HUGO 基本指令

一開始安裝完後可以使用

hugo version

來確認安裝是否成功以己安裝版本為何。

如果要建立一個新的 site，可以使用

hugo new site blog_site_name

這個看到的部落格，很多地方都是顯示 default 的設定，我們可以透過修改設定檔，讓 blog 顯示你要的內容，這邊就需要去修改目錄底下的 hugo.toml 設定檔，上面那一串輸入的指令有 echo "theme = 'ananke'" >> hugo.toml 其實就是加了 theme 設定，指定為 ananke theme。

加入一篇文章

接下來我們先試著新增一篇文章，這樣才算一個完整的 blog，輸入以下命令：

$ hugo new posts/first-post-example.md
Content "myblog/content/posts/first-post-example.md" created

這樣會在主目錄下的 content/posts 目錄裡新增檔案名稱為 first-post-example.md 的文章。而且打開文章一看，hugo 已經幫你生成了部分文章內容，有文章 title，編輯時間，以及是不是還在草稿階段，如下：

+++
title = 'First Post Example'
date = 2023-10-12T11:07:57+08:00
draft = true
+++

接下來你就可以用 markdown 語法寫文章，至於要如何用 markdown 寫文章，網路應該很容易 google 的到，所以就不特別描述了。將剛才的內容改成如下：

+++
title = 'First Post Example'
date = 2023-10-12T11:07:57+08:00
draft = true
+++
